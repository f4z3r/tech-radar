Cloud Governance A framework of rules, policies, and processes that dictates how a company's cloud services are managed and secured.
Container Plattformen Platforms for automating the deployment, scaling, and management of containerized applications.
DAST/SAST DAST (Dynamic Application Security Testing) and SAST (Static Application Security Testing) are methods to find security vulnerabilities in applications.
Data Governance The overall management of the availability, usability, integrity, and security of the data employed in an enterprise.
Data Mesh A decentralized sociotechnical approach to data architecture that enables domain-oriented, self-serve data management.
Data Virtualization A technology that allows applications to retrieve and manipulate data without requiring technical details about the data, such as how it is formatted or where it is physically located.
DevSecOps An augmentation of DevOps to allow for security practices to be integrated into the DevOps approach.
Distributed Integration Platform A platform that enables the connection of various applications, data, and devices across a distributed IT landscape.
Event Driven An architectural pattern in which services are triggered by business events, enabling loosely coupled and scalable systems.
Event Driven API An API designed around the concept of events, where consumers subscribe to be notified when specific events occur.
FinOps A cultural practice and framework that brings financial accountability to the variable spend model of cloud, enabling organizations to get maximum business value.
GraphQL A query language for APIs and a runtime for fulfilling those queries with your existing data.
gRPC A high-performance, open-source universal RPC framework developed by Google.
Hybrid Cloud A cloud computing environment that uses a mix of on-premises, private cloud, and third-party public cloud services with orchestration between the two platforms.
Infrastructure as Code The management of infrastructure (networks, virtual machines, load balancers, and connection topology) in a descriptive model, using the same versioning as DevOps team uses for source code.
Micro Frontends An architectural style where independently deliverable frontend applications are composed into a greater whole.
Microservices An architectural style that structures an application as a collection of loosely coupled services, which implement business capabilities.
Multi Cloud The use of multiple cloud computing and storage services in a single heterogeneous architecture.
Native Cross Plattform The practice of building applications that can run on multiple operating systems with a single codebase, while maintaining a native look and feel.
NoOps The concept that an IT environment can become so automated and abstracted from the underlying infrastructure that there is no need for a dedicated team to manage it.
Pipelines as Code A practice in which the continuous integration and continuous delivery (CI/CD) pipeline is defined as code, allowing it to be versioned and reviewed like any other code.
Policy as Code The practice of managing and automating policies by representing them as code, enabling version control, automated testing, and validation.
Resilient Architecture The ability of a system to withstand and recover from failures, ensuring high availability and reliability.
Serverless A cloud computing execution model in which the cloud provider runs the server and dynamically manages the allocation of machine resources.
Service Mesh A dedicated infrastructure layer that controls service-to-service communication over a network, providing a way to manage and monitor microservices.
Zero Trust Security A security model based on the principle of maintaining strict access controls and not trusting anyone by default, even those already inside the network perimeter.
